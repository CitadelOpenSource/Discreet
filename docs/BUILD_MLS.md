## Step 2: Build WASM Module

```bash
cd discreet-crypto
wasm-pack build --target web --features wasm --no-default-features
```

This produces `discreet-crypto/pkg/` containing:
- `discreet_crypto_bg.wasm` — The compiled WASM binary (~400-500KB gzipped)
- `discreet_crypto.js` — JavaScript bindings (auto-generated by wasm-bindgen)
- `discreet_crypto.d.ts` — TypeScript type definitions

## Step 4: Rebuild the Rust Server

```bash
cargo build
```

The server now has 6 new MLS endpoints:
- `POST /api/v1/key-packages` — Upload KeyPackages
- `GET /api/v1/key-packages/:user_id` — Claim a KeyPackage
- `POST /api/v1/channels/:id/mls/commit` — Submit MLS Commit
- `POST /api/v1/channels/:id/mls/welcome` — Relay Welcome
- `GET /api/v1/channels/:id/mls/info` — Get MLS group info
- `POST /api/v1/identity-keys` — Upload identity keys

## Step 5: Wire WASM into the Client

For the Vite client (`client/`):
```bash
cd client
npm install
npm run dev
```

The WASM module is loaded from `discreet-crypto/pkg/` and exposes:
- `init_identity(userId, username)` — Generate MLS identity
- `generate_key_packages(count)` — Generate pre-keys for server
- `create_group(channelId)` — Create MLS group for a channel
- `add_member(channelId, keyPackageB64)` — Add member to group
- `join_from_welcome(channelId, welcomeB64)` — Join group from Welcome
- `encrypt_message(channelId, plaintext)` — Encrypt with MLS
- `decrypt_message(channelId, ciphertextB64)` — Decrypt with MLS
- `process_commit(channelId, commitB64)` — Process epoch transition
- `self_update(channelId)` — Key rotation for PCS
- `group_info(channelId)` — Get epoch + member count
- `mls_version()` — Get protocol version string

## How Migration Works

Channels have `mls_version`:
- `0` = Legacy PBKDF2 encryption (current, all existing channels)
- `1` = Real MLS RFC 9420 encryption

The client checks `mls_version` per-channel and uses the appropriate encryption.
Channels can be upgraded individually — no big-bang migration needed.
